## Часть 2: Рекомендации для самостоятельной работы

### 2.1. Featured (Избранное) — КРИТИЧНО

Секция Featured — это витрина профиля, первое что видят после Headline. Закрепи:
- 📌 Ссылку на `logic-architecture.com` с описанием
- 📌 Лучшую статью (пост про Obsidian + AnythingLLM набрал комментарии — закрепи)
- 📌 Пост-кейс о каком-нибудь B2B-проекте (если есть)
- 📌 PDF-презентацию Logic Architecture (если создашь)

Как: Профиль → блок «Избранное» → «+» → выбрать публикацию или добавить ссылку.

### 2.2. Переписать Experience — результаты вместо задач

Сейчас описания — списки задач («1. Делал то, 2. Делал это»). Инвесторы и коллеги читают **результаты**. Формула для каждой позиции:

**Было (СБЕР):**
```
1. Разработка чатов для мобильных приложений на основе web-socket.
2. Разработка архитектурного решения микро-сервиса "Чат"...
```

**Стало:**
```
Руководил командой системных аналитиков в Private Banking — мобильные
приложения и веб-сервис для премиальных клиентов банка.

Ключевые проекты:
• Чат-система на WebSocket — спроектировал архитектуру микросервиса,
  провёл через архитектурный комитет, довёл до релиза
• Электронная подпись — разработал архитектуру подтверждения операций,
  интеграция с процессингом (ППРБ, Way 4) и архивными системами
• «Аренда сейфа» — полный цикл: дизайн → архитектура → интеграции →
  релиз. Команда 5 аналитиков, менторство, ежедневные статусы

Стек: BPMN, Jira, Confluence, WebSocket, REST, микросервисы
```

Повтори для каждой позиции: **одно предложение «что делал» → 3-4 bullet с результатами → стек.**
